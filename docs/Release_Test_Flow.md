# Lorana's Dice Manager 发布测试流程

> 目标：在发布前用同一套流程验证“可启动、可部署、可管理、可恢复、可防护”。

## 1. 代码与构建预检

1. 清理临时副本（只删 `.go.纯数字`）：
```powershell
Get-ChildItem -Recurse internal,cmd -File |
  Where-Object { $_.Name -match '\.go\.\d+$' } |
  Select-Object FullName
```
2. 格式化：
```powershell
gofmt -w (Get-ChildItem -Recurse internal,cmd -Filter *.go | ForEach-Object FullName)
```
3. 单元测试：
```powershell
$env:GOCACHE = Join-Path (Get-Location) '.gocache'
go test ./internal/... ./cmd/...
```
4. 构建 Linux 二进制：
```powershell
$env:GOOS="linux"; $env:GOARCH="amd64"; $env:CGO_ENABLED="0"
go build -o build/sealpanel-linux-amd64 ./cmd/ldmanager
```

## 2. 首次启动与登录链路

1. 启动程序，确认自动创建目录：`data/`、`auth/`、`logs/`。
2. 首次访问应跳转创建密码页；创建后可登录。
3. 勾选/不勾选“记住登录”分别验证 Cookie TTL 行为。
4. 重启主程序后验证：
1. 未过期 Cookie 可自动登录。
2. 过期后必须重新登录。

## 3. 面板配置验证

1. 修改监听地址/端口并保存，确认：
1. 配置文件写入成功。
2. 重启后按新地址可访问。
2. 修改 `metrics_refresh_seconds`，确认前端显示与实际刷新频率一致。
3. 修改 `file_manager`、`login_protect` 配置，确认保存后读取回显正确。

## 4. 服务部署与生命周期（每个类型都测）

### 4.1 Sealdice
1. 新建服务：自动下载与手动上传各测一次。
2. 启动参数应为 `--address=0.0.0.0:端口`（仅 Sealdice WebUI）。
3. 验证启动/停止/重启/强制关停按钮。
4. 验证“开机自启”“自动重启（延迟、崩溃次数）”策略。

### 4.2 Lagrange
1. 新建服务后进入配置中心。
2. 验证 HTTP/正向WS/反向WS 开关逻辑与端口保存。
3. 验证签名版本与签名节点列表加载、延迟测试、selected 默认项。
4. 运行中应禁止修改需停机生效项（灰态+提示）。

### 4.3 LLBot
1. 新建服务后验证依赖检测与启动流程日志。
2. QQ 版本管理：默认地址安装、手动地址安装、覆盖安装。
3. 验证二维码预览（`qr-0.png`）可显示。
4. 验证 OneBot 本地监听保持 `127.0.0.1`。

## 5. 端口合法性与占用校验

1. 新建服务时输入非法端口（0、65536、非数字）应实时报错并禁止提交。
2. 输入已被系统进程占用端口应报错并禁止提交。
3. 输入已被本面板服务占用端口应提示“被<显示名>占用”并禁止提交。
4. 修改服务配置时：
1. 未改动端口不应触发占用报错。
2. 改动端口时按上述规则拦截。

## 6. 文件管理验证（服务维度隔离）

1. A 服务进入文件管理后切换到 B，不应继承 A 的路径状态。
2. 上传/下载/删除/复制/粘贴/压缩/解压逐项验证。
3. 文本文件双击编辑：`Ctrl+S` 与保存按钮都生效。
4. 服务运行中应禁用文件管理写操作（按设计约束）。

## 7. 日志与监控

1. 服务日志实时刷新：
1. 不在底部时不强制拉到底。
2. 已在底部时自动跟随。
2. 服务重启后日志应清空并仅显示本次运行。
3. 主程序日志与访问日志写入 `logs/<启动时间>/`。
4. 再次启动时旧日志归档压缩，保留份数符合配置。
5. 总览资源图与快速状态按配置间隔刷新；侧栏可见时也刷新。

## 8. 路由与深链接

1. 直接访问以下 URL 并刷新，页面不应白屏：
1. `/overview`
2. `/services`
3. `/settings`
4. `/services/<id>/overview`
5. `/services/<id>/logs`
6. `/services/<id>/files`
2. 删除当前正在查看的服务后，应自动跳转回服务总览，避免 404 空白页。

## 9. 安全最小检查

1. 未登录访问 `/api/*` 应被拒绝。
2. 登录保护开启后，连续失败触发限流/封禁。
3. 文件管理路径穿越测试：`../`、绝对路径、URL 编码穿越均应被拒绝。
4. 生产部署检查：
1. HTTP 明文风险提示可见。
2. 反代模式按配置处理真实 IP（仅在可信反代下启用）。
3. 云厂商安全组/防火墙端口放行提示可见。

## 10. 回归与发布封板

1. 完整执行一遍“新建→启动→配置→日志→文件→停止→删除”闭环。
2. 运行 30 分钟稳定性观察（重点 LLBot + QQ 注入链路）。
3. 导出最终配置与日志样本，记录版本号、构建时间、提交哈希。
4. 打包发布物：二进制 + 安装脚本 + 默认配置示例 + 变更说明。
